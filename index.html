What is our research about?

Our research integrates semantic web technologies with machine learning to construct models that preserve the intrinsic context and semantics of data. Semantic graph structures ensure that the natural interconnectivity within the data is maintained, facilitating interpretability. This approach diverges from conventional machine learning techniques that typically convert data into numerical forms, potentially stripping it of its original meaning. Ontology

An ontology in the context of computer science and information science is a formal representation of knowledge as a set of concepts within a domain, and the relationships between those concepts.

In our reseach we are using Rdf,Rdfs,and Owl to write our ontologies. We will look at an ontology right after defining these.

Understanding RDF - RDFs - OWL

The Resource Description Framework (RDF), RDF Schema (RDFS), and the Web Ontology Language (OWL) are foundational technologies of the Semantic Web, which is an extension of the World Wide Web through standards set by the World Wide Web Consortium (W3C).

How are they related?

  • RDF provides the basic framework for describing web resources in a triple format.
  • RDFs builds on RDF by providing a basic ontology system to support the definition of classes (types of things) and properties (relationships between things).
  • OWL further extends RDFS by offering a more expressive schema or ontology language. OWL enables more complex relationships between entities, such as equivalence between classes, cardinality (e.g., a person may have multiple email addresses but only one birthdate), and richer property characteristics.

So, basically we use OWL to write our Ontologies in Semantic WorkBench (More of this later).

Let's write our own ontology now:

An ontology to describe Soda and Snack: SodaAndSnackOntology

  • Classes:
    • Product: A general category for items.
      • Drink: A subclass of Product for beverages.
        • CarbonatedDrink: A subclass of Drink specifically for drinks with carbonation.
      • Snack: A subclass of Product for food items intended to be eaten between meals.
  • Properties:
    • hasName: Describes the name of a product. Applies to all Products.
    • isCarbonated: Indicates whether a drink has carbonation. Applies to Drinks.
    • hasFlavor: Specifies the flavor of a product. Applies to both Drinks and Snacks.

Go through the ontology below:

                    
                    
                        Scroll Down to view full code
                        # Namespace prefixes
                        @prefix rdf: <http://www.w3.org/1999/02/22-rdf-syntax-ns#> .
                        @prefix rdfs: <http://www.w3.org/2000/01/rdf-schema#> .
                        @prefix owl: <http://www.w3.org/2002/07/owl#> .
                        @prefix xsd: <http://www.w3.org/2001/XMLSchema#> .
                        @prefix product: <http://example.org/ontology#> .

                        # Class declarations

                        # Product - A generic class for all products.
                        :Product a owl:Class .

                        # Drink - A subclass of Product for drinkable products.
                        :Drink a owl:Class ;
                            rdfs:subClassOf :Product .

                        # CarbonatedDrink - A subclass of Drink for carbonated beverages.
                        :CarbonatedDrink a owl:Class ;
                                        rdfs:subClassOf :Drink .

                        # Snack - A subclass of Product for snack items.
                        :Snack a owl:Class ;
                            rdfs:subClassOf :Product .

                        # Property declarations

                        # hasName - Specifies the name of a product.
                        :hasName a owl:DatatypeProperty ;
                                rdfs:domain :Product ;
                                rdfs:range xsd:string .

                        # isCarbonated - Indicates whether a drink is carbonated.
                        :isCarbonated a owl:DatatypeProperty ;
                                    rdfs:domain :Drink ;
                                    rdfs:range xsd:boolean .

                        # hasFlavor - Describes the flavor of a product.
                        :hasFlavor a owl:DatatypeProperty ;
                                rdfs:domain :Product ;
                                rdfs:range xsd:string .

                        # Instances of classes

                        # CocaColaInstance - An instance of CarbonatedDrink representing Coca-Cola.
                        :CocaColaInstance a :CarbonatedDrink ;
                                        :hasName "Coca-Cola" ;
                                        :isCarbonated true ;
                                        :hasFlavor "Cola" .

                        # LaysClassicInstance - An instance of Snack representing Lays Classic chips.
                        :LaysClassicInstance a :Snack ;
                                            :hasName "Lays Classic" ;
                                            :hasFlavor "Salty" .

                        # SpriteInstance - Another instance of CarbonatedDrink representing Sprite.
                        :SpriteInstance a :CarbonatedDrink ;
                                        :hasName "Sprite" ;
                                        :isCarbonated true ;
                                        :hasFlavor "Lemon-Lime" .

                        # OreoInstance - An instance of Snack representing Oreo cookies.
                        :OreoInstance a :Snack ;
                                    :hasName "Oreo" ;
                                    :hasFlavor "Chocolate" .

Apache Jena

Apache Jena is a free and open-source Java framework for building Semantic Web and Linked Data applications. It provides a programmatic environment for RDF (Resource Description Framework), RDFS (RDF Schema), RDFa, OWL (Web Ontology Language), and SPARQL (SPARQL Protocol and RDF Query Language). Essentially, it's a toolset that allows developers to work with data in RDF format, which is a standard model for data interchange on the Web.
